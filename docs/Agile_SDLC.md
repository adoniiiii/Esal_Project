# Agile и SDLC — Es Al

---

## Методология

Проект ведётся по **Agile/Scrum**, адаптированному под 7-дневный хакатон:
- 2 коротких спринта
- Ежедневные стендапы (10–15 минут каждое утро)
- Ретроспектива в конце каждого спринта
- Все задачи отслеживаются на доске Trello

### Структура Trello доски

```
Backlog  →  In Progress  →  Review  →  Done
```

Дополнительные колонки:
- **Nice to Have** — задачи только если есть время
- **Bugs** — найденные баги по приоритету
- **Decisions Log** — важные технические решения

---

## Спринт 1 (Пн–Ср) — Разработка ядра ✅

**Цель спринта:** Рабочая основа — авторизация, каталог юрт, базовый деплой

| Роль | Задачи |
|---|---|
| PM | Trello заполнен, User Stories написаны, команда синхронизирована, кик-офф проведён |
| Backend | База данных создана, /auth и /places работают, JWT подключён |
| Frontend | React scaffolding, роутинг, экраны: Landing / Login / Catalog |
| AI | Подключение к Grok API, базовый /chat отвечает |
| DevSecOps | GitHub настроен, CI/CD работает, staging задеплоен |

---

### Ретроспектива Спринта 1 (среда вечером — 15 минут)

**3 вопроса для всей команды:**

**1. Что сделали хорошо?**
- Backend и Frontend в первый день согласовали формат API — конфликтов не было
- DevSecOps поднял staging окружение уже во вторник
- Ежедневные стендапы держали команду в курсе

**2. Что замедляло работу?**
- Подключение Grok API заняло больше времени чем планировалось
- Frontend во вторник ждал эндпоинт /places от Backend

**3. Что меняем в Спринте 2?**
- Backend сразу пишет в общий чат когда эндпоинт готов
- AI интеграция начинается раньше, не в конце дня

---

## Спринт 2 (Чт–Пт) — Интеграция и полировка 🔥

**Цель спринта:** Связать всё вместе — бронирование работает, ИИ отвечает с контекстом юрты, фронт общается с бэком

| Роль | Задачи |
|---|---|
| PM | Полная документация, презентация, координация финальной сборки |
| Backend | Логика /bookings, /chat с контекстом юрты, исправление багов |
| Frontend | Форма бронирования, компонент AI-чата, полная интеграция с API |
| AI | Контекстные промпты с данными юрты, история диалога, тестирование качества ответов |
| DevSecOps | Rate limiting, финальный прод-деплой, проверка безопасности |

---

### Ретроспектива Спринта 2 (пятница вечером — 15 минут)

**3 вопроса для всей команды:**

**1. Что сделали хорошо?**
- Все 4 MVP-функции работают и связаны
- Grok API в итоге заработал, чатбот отвечает с контекстом юрты
- Прод-деплой стабилен

**2. Что замедляло работу?**
- Форма бронирования и бэкенд изначально использовали разные форматы дат — исправили в четверг
- Проблема с Grok API в начале спринта заблокировала AI-разработчика на несколько часов

**3. Что сделали бы иначе в следующий раз?**
- В первый день зафиксировать формат запросов и ответов API в общем документе
- Иметь запасной вариант для ИИ на случай если основной API не работает

## Decisions Log — Важные решения

| Дата | Решение | Причина |
|---|---|---|
| Пн | Тип объекта: yurt / topchan через CHECK constraint | Проще чем отдельная таблица для MVP |
| Пн | Владельцы юрт не регистрируются — данные вносит admin | MVP только для туристов, контроль качества на старте |
| Вт | JWT для авторизации (не сессии) | Stateless, удобнее для React |
| Вт | Grok API для чатбота | Быстрый отклик, доступный тариф |
| Ср | ON DELETE RESTRICT в bookings | Защита от удаления пользователя с активными бронями |
| Чт | Передавать данные юрты в контекст промпта | Более точные и релевантные ответы чатбота |
| Чт | Временный мок-ответ пока Grok API не работал | Не блокировать Frontend пока AI решает проблему |

## Фазы SDLC

| Фаза | Дни | Что делали | Артефакты |
|---|---|---|---|
| Planning | Пн утро | Определили MVP, создали Trello, распределили роли | Trello доска, список задач |
| Requirements | Пн | User Stories, CJM, персона пользователя | User Stories, CJM |
| System Design | Пн–Вт | Архитектура, схема БД, API контракт | Диаграмма, SQL схема, список эндпоинтов |
| Implementation | Пн–Ср | Ядро: авторизация, каталог, базовый ИИ, CI/CD | Код в GitHub, ежедневные коммиты |
| Integration | Чт–Пт | Связка всех частей: бронирование, ИИ с контекстом | Задеплоенное приложение |
| Testing | Вс | Полный QA флоу, исправление багов, проверка безопасности | Список багов в Trello |
| Deployment | Вс–Пн | Финальный прод-деплой, чек-лист, подготовка к демо | Рабочая ссылка, презентация |
